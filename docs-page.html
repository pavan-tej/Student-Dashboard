Introduction Last updated: 2021-05-06

Student Dashboard helps students get access to their daily necessary needs like timetable, track their attendence and academic performance, share study materials, post complaints, get updates about internship and research opportunities within and outside the institute. It also helps students to schedule their daily activities ( like a To-do app ).

You can visit our website here

Features

We have developed various features for students to track their performance and made things easiler that reduce a huge of work. Why still waiting let's explore all of them.


Attendence

Attendence helps students to track their subject-wise attendence updated on a daily basis.

Timetable

Shows time table of that particular student when logged in using his credentials. So that student cannot miss the class.

Study Materials

Students can find notes, Academic Resources, Previous Year's Question Papers here.

Almanac

Almanac for that particular semester is shown in this section.

Student requests

Students can request academic office for grade sheets, certificates and any other things.

Complaints

Students can raise complaints directly and makes it easy for resolving them.

Library

Here student can find the list of available books in the institute's library.

Opportunities Platform

Information regarding Internships, research, jobs will be posted here.

Alumni Network

Students can reach out to alumni for placement related guidance and queries.

Installation

Let us learn how to install the application and run it in your local machine.

Prerequisite

To install the application you require the following softwares.

Git Node.JS npm

Starting the installation

Now let us the installation.

Steps as follows:
  • Let us clone the repository:
    git clone https://github.com/saisatwik99/Student-Dashboard
  • Navigate to the above folder:
    cd Student-Dashboard
  • Install node modules:
    npm install
  • In package.json you can find the follow code.
  • If the above step is successfull. Now create a file with name .env and insert your variables
    MONGODB_URI, TOKEN_SECRET, PORT
  • We reached the final step, Lets run the app.
    npm run dev

APIs

Let us understand the folder structure and how APIs are written.

Folder Structure:
controller All the api's and their functionalities are written here.
models Here we structure the schemas of different models. Ex: user.
node_modules This folder contains all the packages we are using to run the application.
public It contains the css, images of the web pages.
validation In validation we validate the data and information.
view In this folder it contains all ejs files for the web pages.
index.js The main file where application starts running.
package-lock.json Automatically generated file to operate node modules.
package.json Package.json specify the version of a package that our app depends on.

Our application is written in express.js . We use apis to write the function. Used MongoDB to store the data. We send request to send and receive data.
